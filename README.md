# Очистка и разведочный анализ данных для транзакций продаж в кофе

Проект по очистке и разведочному анализу данных (EDA) на примере датасета с транзакциями продаж в кофейне. 

Цель проекта — продемонстрировать полный цикл работы с "грязными" данными: от предобработки до визуализации и выводов.

Грязный датасет взят [отсюдова](https://www.kaggle.com/datasets/ahmedmohamed2003/cafe-sales-dirty-data-for-cleaning-training)

## Шаги, сделанные для очищения данных:
1. Заменены недопустимые записи, такие как "UNKNOWN" и "ERROR", на Nan.
2. Заменил пропущенные числовые значения на **медиану**.
3. Заменил пропущенные категориальные значения на **моду**.
4. Установил для каждой столбца нужный тип данных, так как в грязных данных они имели все объект object.
5. Feauture Engineering: Добавил новый столбец "Transaction month".
